# 🎬 Emby-Cloudflare-Proxy 

基于 Cloudflare Workers 打造的 Emby/Jellyfin 轻量级智能反向代理工具。
提升反代性能与访问安全性，添加定时播报功能。

## ✨ 核心特性

### 1. 🔀 智能寻址与重组
- **万能拼接**：支持直接在浏览器/客户端输入 `https://代理域名/https://源站:端口` 进行访问。
- **端口自动找回**：自动从 Request 头部提取并继承目标端口，彻底解决 `521 Web server is down` 报错（适配部分官方客户端自动截断端口的 Bug）。

### 2. 🧠 智能分级缓存系统
- **海报秒开**：自动嗅探 Emby 刮削完成特征码（tag=），将海报在 CDN 边缘节点强力缓存长达 1 年。
- **黑图放行**：针对源站未刮削完毕的临时占位图，仅执行 2 小时微缓存，确保服务器刮削完成后客户端能及时更新。
- **API 丝滑缓存**：对 Resume 等高频读取接口进行 10 秒微缓存，有效解决频繁进退详情页时的加载卡顿问题。

### 3. 🛡️ 访问控制与监控播报
- **Web 控制台**：基于 Cloudflare KV 的轻量化后台面板，支持在线实时修改配置。
- **强迫症白名单**：支持黑/白名单双模式防御。输入框兼容 `# 中文备注` 及空行排版，并能自动提取混合链接中的纯净域名。
- **TG 定时播报**：每日定时推送 Cloudflare 额度消耗报表，让你随时掌握系统负载。

## 🚀 部署方式

### 方式一：GitHub Actions 自动化部署 (推荐)
通过 GitHub Actions 配置一次环境变量，后续即可全自动推送到 Cloudflare 边缘节点。

#### 步骤 1：准备 KV 数据库
1. 登录 Cloudflare，进入【Workers 和 Pages】→ 【KV】。
2. 创建新命名空间（例如：`emby_kv`），复制右侧的 **命名空间 ID**。

#### 步骤 2：Fork 本仓库并配置 Secrets
1. Fork 本项目到你的 GitHub 账户。
2. 进入仓库的 `Settings -> Secrets and variables -> Actions`。
3. 点击 `New repository secret`，添加以下 3 个必需变量：

| Secret 变量名 | 填写说明 | 获取位置 |
|--------------|----------|----------|
| `CF_ACCOUNT_ID` | 你的 Cloudflare 账户 ID | CF 网页右下角的“帐户 ID” |
| `CF_API_TOKEN` | 具备编辑 Worker 权限的令牌 | CF 右上角头像 → 我的个人资料 → API 令牌 |
| `CF_KV_ID` | 步骤 1 中创建的 KV 空间 ID | CF 控制台 → KV → 对应命名空间右侧的 ID |

---

### 方式二：CF 网页手动部署
直接在 Cloudflare 网页端手动创建并部署。

#### 步骤 1：创建 KV 数据库
1. 进入 Cloudflare 左侧菜单 → 【Workers 和 Pages】→ 【KV】。
2. 点击【创建命名空间】，自定义名称（例如：`emby_lite_kv`）。

#### 步骤 2：部署 Worker 代码
1. 进入【Workers 和 Pages】→ 【概述】→ 【创建 Worker】，命名并完成初始部署。
2. 点击【编辑代码】，粘贴本仓库 `_worker.js` 的完整代码覆盖原有内容，点击部署。

#### 步骤 3：绑定环境变量 (关键)
**绑定 KV 命名空间**
- 进入 Worker 【设置】→ 【绑定】→ 【KV 命名空间绑定】。
- 变量名称：**必须填写小写的 `kv`**。
- 命名空间：选择步骤 1 中创建的数据库。

**设置后台路径**
- 进入 Worker 【设置】→ 【变量和机密】→ 添加变量。
- 变量名：`admin`
- 变量值：自定义后台路径（如 `/lite-admin`），不设置则默认入口为 `/emby-admin`。

#### 步骤 4：绑定自定义域名
在 Worker 【设置】→ 【触发器】→ 【自定义域】中，绑定托管在 Cloudflare 上的域名即可。

> ⚠️ 初始默认密码：`admin`（登录后台后请立即修改密码以保证安全）

## 📱 配置 Telegram 定时播报 (可选)
Telegram 的推送配置已完全集成至 Web 控制台，无需在代码中硬编码。

1. **填写推送密钥**：访问你的后台管理页面，在【Telegram 监控报表通道】区块中，填入你的 `TG Bot Token` 和 `TG Chat ID`，点击保存。
2. **设置触发器 (Cron)**：进入 Cloudflare 的 Worker 管理页面，切换到【触发器】选项卡。
3. **添加 Cron 表达式**：在“Cron 触发器”处点击添加，填入 `0 12 * * *`（此时间对应北京时间每天晚上 20:00），系统即会在该时间准时为你推送今日请求用量报表。

## 📖 客户端使用示范
部署完成后，在 Emby/Jellyfin 客户端按以下格式添加服务器：

| 字段         | 填写示例                          |
|--------------|-----------------------------------|
| 服务器地址   | `https://你的代理域名.com/https://目标服主地址:8443` |
| 端口         | `443`（无论源服端口是多少，外层统一填 443） |
| HTTPS        | 勾选开启                          |
